# Changelog

## 0.1.3-alpha

- Добавлена быстрая диагностика `TMDB/Lampa API` перед массовым сопоставлением.
- Добавлены кнопки:
  - `Проверить TMDB/Lampa API`
  - `Показать диагностику TMDB/Lampa`
- Таймаут TMDB/Lampa-запроса уменьшен до 8000 мс, чтобы не ждать по 30 секунд на каждую карточку.
- Массовое сопоставление теперь сначала делает preflight-запрос по тестовому IMDb `tt0111161`.
- Если preflight не отвечает, сопоставление не запускает 100+ долгих запросов и сохраняет диагностический отчёт с `aborted: true`.
- Если во время сопоставления накопилась серия ошибок TMDB/Lampa API, проверка останавливается досрочно.
- Лимит сопоставления рекомендуется начинать с 10/25, затем увеличивать при успешном preflight.
- Импорта по-прежнему нет: версия только read-only.

## 0.1.2-alpha

- Уточнена терминология отчёта готовности импорта.
- Добавлены `importCandidates` и `blockedItems`.
- Успешный статус: `matched_import_candidate`.
- Kinopoisk-only явно блокируется как `blocked_kinopoisk_only` до появления надёжного Kinopoisk -> TMDB/Lampa resolver.
