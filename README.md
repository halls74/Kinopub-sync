# KinoPUB Sync v0.0.7-alpha

Read-only диагностическая сборка Lampa-плагина для проверки токена KinoPUB, чтения папок/закладок и безопасной локальной очистки следов старого Sync в Lampa.

## Что делает

- Проверяет наличие и работоспособность токена основного KinoPUB-плагина.
- Читает папки и закладки KinoPUB через API `/v1/bookmarks` и `/v1/bookmarks/<folder_id>`.
- Использует пагинацию `page + perpage=50`.
- Формирует подробный JSON-отчёт по папкам, типам контента, внешним ID и повторам.
- Разделяет счётчики:
  - `rawRowsFetched` — все строки, физически полученные от API, включая повтор хвоста;
  - `acceptedRawRows` — принятые неповторные строки;
  - `repeatedTailRows` — повторившиеся строки последней страницы;
  - `missingAcceptedByCount` — недостача относительно счётчика KinoPUB по принятым строкам.
- Содержит блок в основных настройках **Очистка данных** внутри настроек основного плагина.
- Подраздел очистки ищет и удаляет только локальные следы старого Sync в Lampa: закладки, папки `KinoPUB: ...`, историю, продолжение просмотра/timeline и служебные `kp_sync_*` карты.

## Что удалено из этой версии

- Проверка доступности карточек через `/v1/items/<id>?nolinks=1` убрана из интерфейса и публичного API плагина.
- WEB-проверка `https://kino.pub/item/view/<id>` не выполняется, потому что страница зависит от web-авторизации KinoPUB/cookies и может быть недоступна из Lampa.

## Что версия НЕ делает

- Не импортирует данные в Lampa.
- Не создаёт тестовые папки.
- Не синхронизирует историю или таймкоды с KinoPUB.
- Не проверяет WEB status code страниц KinoPUB.
- Не меняет данные в KinoPUB.

## Почему счётчик закладок отличается от количества проверяемых/уникальных карточек

KinoPUB может показывать количество строк закладок по папке, например `939`, но в отчёте дополнительно есть уникальные `item id`. Один и тот же фильм/сериал может быть в нескольких папках, а API может повторять последнюю страницу. Поэтому:

- `countReported` — счётчик строк в папке по KinoPUB;
- `acceptedRawRows` — реально принятые строки списка;
- `uniqueItems` — уникальные `item id` внутри папки;
- `uniqueGlobalItems` — уникальные `item id` по всем папкам.

## Настройки

Основной раздел:

`Настройки → KinoPUB Sync 0.0.7`

Кнопки:

- Проверить токен основного KinoPUB
- Считать папки и закладки KinoPUB
- Показать краткий отчёт
- Скопировать полный отчёт
- Очистить отчёт
- Очистка данных

Кнопка **Очистка данных** открывает блок в основных настройках с действиями:

- Найти старые данные Sync в Lampa
- Показать отчёт очистки Lampa
- Скопировать отчёт очистки Lampa
- Очистить найденные старые данные Sync

## Установка

Если основной KinoPUB-плагин уже установлен и авторизован:

```text
https://<host>/docs/kp-sync.js?v=007
```

Если нужен loader, который сначала загружает оригинальный `lampa_kinopub`, а потом KinoPUB Sync:

```text
https://<host>/docs/kp.js?v=007
```

## Тестирование

1. Установить URL `kp-sync.js?v=007`.
2. Открыть `Настройки → KinoPUB Sync 0.0.7`.
3. Нажать `Проверить токен основного KinoPUB`.
4. Нажать `Считать папки и закладки KinoPUB`.
5. Скопировать полный отчёт и проверить `countReported`, `acceptedRawRows`, `repeatedTailRows`, `missingAcceptedByCount`.
6. Для очистки открыть кнопку `Очистка данных`, сначала выполнить сканирование, изучить отчёт и только затем запускать удаление.
