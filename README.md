# KinoPUB Sync v0.1.0-stable

Стабильная read-only сборка для поэтапной разработки.

## Что делает эта версия

- Использует токены основного `lampa_kinopub`: `kp_token` и `kp_refresh`.
- Проверяет доступность KinoPUB API.
- Читает доступные папки и закладки KinoPUB через read-only запросы.
- Читает большие папки основным способом `/v1/bookmarks/<folder_id>?page=N&perpage=50`.
- Формирует более честную диагностику хвоста пагинации: `rawRowsFetched`, `acceptedRawRows`, `repeatedTailRows`, `missingAcceptedByCount`.
- Показывает статистику по всем встреченным `type/subtype`, а не только `movie/serial`.
- Добавляет отдельную read-only проверку доступности карточек через `/v1/items/<item_id>?nolinks=1`.
- Ускоряет проверку доступности: карточки проверяются небольшими параллельными батчами.
- В отчёт доступности добавляет проблемные карточки, причины ошибки и ссылки:
  - `https://kino.pub/item/view/<id>`
  - `https://www.imdb.com/title/tt.../`, если есть IMDb ID
  - `https://www.kinopoisk.ru/film/<id>/` или `/series/<id>/`, если есть Kinopoisk ID
- Добавляет отдельный подраздел настроек **KinoPUB Sync — Очистка данных**.
- В очистку данных входят старые локальные следы Sync:
  - некорректные закладки;
  - папки `KinoPUB: ...`;
  - история Lampa (`favorite.history`);
  - продолжение просмотра / timeline, если запись связана со старыми `kp_sync_*` картами;
  - старые служебные карты `kp_sync_identity_map`, `kp_sync_episode_map`, `kp_sync_timeline_map`, snapshot-карты.

## Важно про 939 и 925

`countReported` — это счётчик строк закладок, который сообщает KinoPUB для папки.  
`uniqueGlobalItems` — это уникальные item id, которые реально пришли из API и были дедуплицированы между папками.

Поэтому число проверяемых карточек доступности может быть меньше количества закладок: проверка `/v1/items/<id>` выполняется только по уникальным item id, реально полученным в отчёте чтения.

## Чего версия не делает

- Не авторизуется отдельно.
- Не импортирует закладки в Lampa.
- Не создаёт тестовую папку импорта.
- Не синхронизирует историю и таймкоды.
- Не выполняет экспорт Lampa → KinoPUB.
- Не удаляет и не меняет данные в KinoPUB.
- Не проверяет WEB status code `https://kino.pub/item/view/<id>`, потому что эта страница зависит от web-авторизации и может быть недоступна из Lampa/CORS.

## Настройки в Lampa

`Настройки → KinoPUB Sync 0.1.0`

Основные действия:

- Проверить токен основного KinoPUB
- Считать папки и закладки KinoPUB
- Показать краткий отчёт
- Скопировать полный отчёт
- Очистить отчёт

Проверка доступности карточек:

- Лимит проверки карточек: `0` означает проверить все уникальные карточки из последнего отчёта
- Проверить доступность карточек KinoPUB
- Показать отчёт доступности
- Скопировать отчёт доступности
- Очистить отчёт доступности

`Настройки → KinoPUB Sync — Очистка данных`

- Найти старые данные Sync в Lampa
- Показать отчёт очистки Lampa
- Скопировать отчёт очистки Lampa
- Очистить найденные старые данные Sync

Очистка затрагивает только локальные Lampa-данные с признаками старого Sync-импорта, старые `kp_sync_*` карты и папки `KinoPUB: ...`. KinoPUB не меняется.

## Установка

Если основной KinoPUB-плагин уже установлен:

```text
https://<host>/docs/kp-sync.js?v=010
```

Если нужен загрузчик, который сначала подтянет основной `lampa_kinopub`:

```text
https://<host>/docs/kp.js?v=010
```

## Как тестировать v0.1.0

1. Нажать `Считать папки и закладки KinoPUB`.
2. Скопировать полный отчёт и проверить `acceptedRawRows`, `repeatedTailRows`, `missingAcceptedByCount`.
3. Для короткого теста поставить лимит проверки карточек `50` или `100`.
4. Нажать `Проверить доступность карточек KinoPUB`.
5. Если короткий тест корректен, поставить лимит `0` и проверить все уникальные карточки.
6. В подразделе `KinoPUB Sync — Очистка данных` сначала нажать `Найти старые данные Sync в Lampa`.
7. Проверить отчёт очистки и только после этого нажать `Очистить найденные старые данные Sync`.
