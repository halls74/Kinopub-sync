# KinoPUB Sync v0.1.2-alpha

Read-only alpha-сборка для Lampa.

## Назначение

v0.1.2 продолжает этап `0.1.x`: read-only проверка готовности будущего импорта KinoPUB → Lampa/TMDB без записи в Lampa и без изменения KinoPUB.

## Что делает

- Проверяет токен основного KinoPUB-плагина (`kp_token` / `kp_refresh`).
- Читает папки и закладки KinoPUB через API.
- Показывает диагностические поля пагинации: `acceptedRawRows`, `repeatedTailRows`, `missingAcceptedByCount`.
- Показывает реальные повторяющиеся `item id` внутри папки в `duplicateItemsInFolder`.
- Выполняет read-only проверку готовности импорта: уникальные KinoPUB-карточки сопоставляются с TMDB/Lampa по IMDb через `Lampa.TMDB.api('find/<imdb>?external_source=imdb_id')`.
- Делит результаты на `importCandidates` и `blockedItems`: импорт-кандидаты появляются только если получена настоящая TMDB/Lampa-карточка; Kinopoisk-only блокируется до отдельного надёжного резолвера.
- Оставляет локальную очистку старых данных Sync: закладки, история, продолжение просмотра и служебные карты.

## Что НЕ делает

- Не импортирует закладки в Lampa.
- Не меняет закладки KinoPUB.
- Не выполняет WEB-проверку `kino.pub/item/view/...`.
- Не выполняет ложное точное сопоставление по Kinopoisk ID через TMDB: стандартный путь TMDB Find используется по IMDb. Kinopoisk ID пока сохраняется как отдельный диагностический резерв для будущего резолвера.
- Не выполняет title-based fallback matching, чтобы не получить ложные совпадения.

## Настройки

`Настройки → KinoPUB Sync 0.1.2`

Основные действия:

- Проверить токен основного KinoPUB
- Считать папки и закладки KinoPUB
- Показать краткий отчёт
- Скопировать полный отчёт
- Очистить отчёт

Сопоставление:

- Лимит сопоставления
- Сопоставить карточки с TMDB/Lampa
- Показать отчёт сопоставления
- Скопировать отчёт сопоставления
- Очистить отчёт сопоставления

Очистка данных:

- Найти старые данные Sync в Lampa
- Показать отчёт очистки Lampa
- Скопировать отчёт очистки Lampa
- Очистить найденные старые данные Sync

## Установка

Прямой URL:

```text
https://<host>/docs/kp-sync.js?v=012
```

Loader:

```text
https://<host>/docs/kp.js?v=012
```

## Рекомендуемый порядок проверки

1. Установить URL `kp-sync.js?v=012`.
2. Открыть `Настройки → KinoPUB Sync 0.1.2`.
3. Нажать `Считать папки и закладки KinoPUB`.
4. Проверить `duplicateItemsInFolder`, особенно после ручной очистки дублей на портале.
5. Поставить `Лимит сопоставления = 50` или `100`.
6. Нажать `Сопоставить карточки с TMDB/Lampa`.
7. Скопировать отчёт сопоставления.
8. Если короткий тест нормальный, поставить лимит `0` и проверить все уникальные карточки.

## Статусы проверки готовности импорта

Отчёт сохраняется в `kp_sync012_match_report` и содержит:

- `matched_import_candidate` — получена настоящая TMDB/Lampa-карточка по IMDb и ожидаемому типу `movie`/`tv`; такую карточку можно рассматривать как кандидата для будущего импорта. Внутри результата есть `import_allowed: true` и `import_card` с TMDB id/type.
- `blocked_kinopoisk_only` — у KinoPUB-карточки нет IMDb, но есть Kinopoisk ID; без отдельного Kinopoisk → TMDB/Lampa resolver автоматический импорт запрещён.
- `blocked_no_external_id` — нет ни IMDb, ни Kinopoisk ID; автоматический импорт запрещён.
- `blocked_no_tmdb_result` — TMDB Find по IMDb не вернул movie/tv; автоматический импорт запрещён.
- `blocked_kind_mismatch` — TMDB Find вернул другой тип `movie/tv`; автоматический импорт запрещён.
- `blocked_ambiguous` — TMDB Find вернул несколько результатов ожидаемого типа; автоматический импорт запрещён до ручного/дополнительного выбора.
- `blocked_api_error` — ошибка или таймаут Lampa/TMDB API; автоматический импорт запрещён до повторной проверки.

## Важно

v0.1.2-alpha — это не импорт, а проверка готовности будущего импорта. Даже при `matched_import_candidate` плагин ничего не записывает в `favorite`, `history`, `timeline` или `custom_favorite`. Для будущего импорта разрешены только результаты из `importCandidates`; всё из `blockedItems` автоматически импортировать нельзя.
